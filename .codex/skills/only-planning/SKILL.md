# Only Planning

当这个 skill 被使用时，只进行分析和方案说明，不执行任何实际操作。

## 核心规则

- 不修改任何文件。
- 不创建、删除、移动文件或目录。
- 不运行命令、程序、脚本、测试、构建、格式化工具或服务。
- 不安装依赖。
- 不调用会修改外部系统、仓库、issue、PR 或评论的工具。
- 只给出文字分析、解决思路和可选方案。

## 回答方式

- 先说明对问题的理解。
- 优先给出一个推荐方案。
- 如果有必要，可以再给出 2 到 3 种备选方案。
- 简短说明每种方案的优缺点和适用场景。
- 说明关键风险、前置条件或假设。
- 可以给出示例代码、命令、文件路径或 patch 思路，但只能作为文本示例，不能实际执行。
- 如果信息不足，先提出简短的问题确认。

## 退出规则

除非用户明确要求“开始修改”“帮我执行”“直接创建”“运行一下”“应用这个方案”等，否则一直保持只分析、不操作。
